Characterization Pipeline

XRD Batch Analysis
to Technical Memo

Automated peak fitting, crystallite sizing, and anomaly detection across 8 XRD diffraction patterns — delivered as a formatted, portable HTML memo.

The before / after

4 hrs
Manual workflow Open each file in JADE or HighScore. Fit peaks by hand. Record FWHM to spreadsheet. Compute Scherrer sizes. Write memo.
→
~90 sec
This pipeline Load → Fit → Analyze → Report. Single Python call. Zero manual steps. Portable HTML memo out.

What it does

  • 1
    Load

    Parses RRUFF-format .txt files with variable-length metadata headers. Reads wavelength from file; defaults to Cu Kα (1.54056 Å).

  • 2
    Fit

    Auto-detects peaks via scipy. Builds a dynamic lmfit composite model — one PseudoVoigt profile per peak. Computes R², AIC, BIC per sample.

  • 3
    Analyze

    Extracts d-spacings (Bragg's law) and crystallite sizes (Scherrer equation, K=0.9). Flags outliers by Z-score and IQR.

  • 4
    Report

    Renders a Jinja2 HTML memo with all 8 sample figures embedded as base64 PNGs. Single portable file — no external dependencies.

The gap this fills

SpectraFit, xrdfit, and Materials_Data_Analytics all stop at fit parameters. Every one delivers a table of numbers. Nobody generates the memo. This pipeline goes the last mile — from raw diffraction data to a formatted deliverable a non-specialist buyer can read and act on.

Technical approach

Peak fitting

lmfit PseudoVoigt (Gaussian + Lorentzian mix). Auto-detected peak count; capped at 15 per spectrum.

Crystallite size

Scherrer equation D = Kλ/βcosθ, K=0.9. Upper-bound estimates; instrument broadening not subtracted.

Outlier detection

Dual method: Z-score (|z| > 2.5) + IQR supplement. IQR is more reliable at small n.

Output

Single-file HTML memo. All figures base64-embedded. Print-ready layout with summary table and next steps.
