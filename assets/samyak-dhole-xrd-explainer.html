The Problem

Diffraction data from a same-material batch, whether an annealing series, a processing sweep, or a QC run, is rarely reported consistently. The standard workflow opens each pattern separately in JADE, HighScore, or EVA, fits peaks by hand or with instrument software defaults, copies FWHM and position values into a spreadsheet, and computes Scherrer sizes with a formula cell. Each analyst applies slightly different ROI widths, background subtraction choices, and quality judgements. The resulting table is correct for that batch, but the approach does not transfer: the next batch requires the same manual effort, and comparing across batches depends on whether the same choices were applied.

For a characterization team generating routine series: annealing ramps, composition gradients, stability runs, this is not a one-time cost. It is the cost of every batch. The scientific content of the analysis (which peaks to fit, how to handle overlaps, what constitutes a poor fit) is not the bottleneck. The reporting burden is.

The Pipeline

The pipeline takes a directory of RRUFF-format .txt files, identifies peaks automatically, fits them with a physically motivated profile model, extracts batch-level process signals, and renders a single portable HTML memo. The analyst receives a reviewable deliverable, not a table of numbers.

Peak detection

Auto-detection uses a Difference of Gaussians (DoG) approach, following the method described in xrfit: a narrow Gaussian (σ≈0.05°) preserves sharp peaks while a wide one (σ≈0.3°) approximates the local background. Peaks are candidates where the difference exceeds an adaptive threshold based on the median absolute deviation (MAD) of the noise floor. This reduces hand-tuning: the same parameter set applies across patterns from the same instrument and material family without per-sample adjustment.

Fitting

Each candidate peak gets a local ROI (±0.25° around the detected centroid) and is fitted with an lmfit PseudoVoigt profile plus a linear local background. PseudoVoigt mixes Gaussian and Lorentzian character via a mixing parameter η, which the optimizer determines from the data. Levenberg-Marquardt minimization converges in roughly 30 ms per peak. Fits with R² below threshold are flagged for manual review rather than silently included in the batch summary.

Process signal extraction

For a same-material batch the scientifically meaningful output is not the fit parameters themselves but the trends across samples: is the dominant reflection drifting in position (lattice parameter change)? Is FWHM broadening (crystallite refinement, increasing disorder)? Are Scherrer sizes falling monotonically or scattering? The pipeline extracts these signals from the dominant peak of each sample and presents them as a decision layer alongside the fit-quality table.

Output

The memo is a single self-contained HTML file: no external dependencies, no server required, prints cleanly. It includes a process signal section, a decision implication summary, validation notes, a per-sample summary table, a full per-peak table, and base64-embedded fit-overlay figures for each sample.

Results: Quartz Annealing Series

8 samples · 14–16 peaks per pattern · no flags raised Dominant reflection at 26.65° 2θ (quartz 101 reflection, Cu Kα, ICDD 00-046-1045). Position stable across the series. FWHM broadens from 0.0794° to 0.2205° over 8 samples. Scherrer upper-bound crystallite size decreases from 102.8 nm to 37.0 nm.

The FWHM trend and corresponding Scherrer size decrease are consistent with progressive crystallite refinement under the annealing conditions encoded in the synthetic series. The dominant peak position holds at 26.65° throughout, indicating no measurable lattice parameter drift. No automated R² or IQR flags were raised across the 8 samples, so the memo goes directly to scientist review without exception handling.

Limitations

Scherrer sizes are upper-bound estimates. Instrument broadening (from beam optics, sample geometry, and detector resolution) is not subtracted. The pipeline reports these as trend indicators within a batch, not as absolute crystallite dimensions.
Designed for same-material batches. Peak detection and process signal extraction assume a shared dominant reflection across samples. Mixed-phase or highly variable patterns require per-sample phase assignment and are outside this pipeline's current scope.
Flags identify, not diagnose. R² and IQR flags mark samples that warrant manual review. The pipeline does not interpret the cause of a poor fit: that remains scientist work.
File format. The current loader handles RRUFF-format .txt files. Other common formats (XRDML, UXD, raw binary) require a format-specific loader, which is straightforward to add but not included in this demo.
